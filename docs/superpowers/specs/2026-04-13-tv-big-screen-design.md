# TV 大屏重排设计

## 背景

当前 `tv.html` 在电视全屏浏览器中存在三类展示问题：

- 页面外层仍保留较宽安全边距，视觉上没有“铺满屏幕”。
- 人员页卡片采用纵向堆叠，横向空间利用率低，右侧留白明显。
- 多处文字层级偏小，远距离观看时显得不够饱满。

## 目标

- 将 TV 页面调整为近乎满屏展示，只保留极窄安全边距。
- 将人员卡片改成左右分区结构，让详情内容占据卡片右半区。
- 提升标题、指标、详情等关键文字的字号与层级，增强电视场景可读性。

## 方案

### 1. 大屏容器

- `page-tv` 去掉固定最大宽度，改为直接占满视口宽度。
- `tv-shell` 缩小外层边距和圆角，增加内部可用高度。
- 同步扩大头部和内容区的布局比例，让主内容更贴近 16:9 大屏使用方式。

### 2. 人员卡片

- 保留外层两列卡片布局，避免同屏人数骤减。
- 每张卡片内部改为：
  - 左侧：姓名、状态、项目、时间范围
  - 右侧：工作分工、本周计划
- 详情信息做成独立信息区块，减少右侧空白并突出计划内容。

### 3. 字体层级

- 提升以下区域字号：
  - 主标题、屏幕标题
  - 指标卡数字与标签
  - 人员姓名、项目标签、时间标签
  - 工作分工、本周计划正文
- 通过 `clamp()` 保持在 1080p 电视与较小窗口下都不过度失控。

## 风险与处理

- TV 页是静态页面，无现成前端测试框架。为降低回归风险，新增一个基于 Node 内置测试器的最小回归测试，验证人员卡片 HTML 结构。
- 移动端不适合沿用大屏双栏卡片，因此保留媒体查询回退为单列布局。

## 验证

- 运行 Node 测试，确认人员卡片结构按预期输出。
- 本地启动服务并检查 `tv.html` 页面是否仍能正常加载。
- 人工核对三项结果：
  - 外层是否接近铺满屏幕
  - 人员卡片右侧是否被有效利用
  - 详情字体是否明显放大
